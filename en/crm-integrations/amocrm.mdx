# AmoCRM Integration

Integrate Gridix with AmoCRM to automatically send leads to your CRM system. This eliminates manual data entry and ensures all leads are captured in your sales pipeline.

## Overview

The AmoCRM integration allows you to:

- **Automatically send leads** to AmoCRM when customers submit inquiries
- **Configure pipelines** and statuses for lead routing
- **Set responsible users** for automatic assignment
- **Track sync status** for each lead
- **View leads in AmoCRM** directly from Gridix

<Check>
Benefits of AmoCRM integration:
- No manual data entry
- Real-time lead synchronization
- Automatic pipeline assignment
- Centralized lead management
- Reduced data loss
</Check>

## Getting Started

### Prerequisites

Before connecting AmoCRM, ensure you have:

- An active AmoCRM account
- Admin or integration access to AmoCRM
- A Gridix project with leads enabled
- Pro subscription (for CRM integrations)

## Connecting AmoCRM

### Step 1: Initiate Connection

<Steps>
  <Step title="Navigate to Settings">
    Go to your project settings or admin dashboard.
  </Step>
  
  <Step title="Open CRM Integration">
    Click on "CRM Integration" or "AmoCRM" section.
  </Step>
  
  <Step title="Start Connection">
    Click "Connect AmoCRM" button.
  </Step>
</Steps>

### Step 2: Authorize Access

<Steps>
  <Step title="Redirect to AmoCRM">
    You'll be redirected to AmoCRM authorization page.
  </Step>
  
  <Step title="Login to AmoCRM">
    Log in to your AmoCRM account if not already logged in.
  </Step>
  
  <Step title="Grant Permissions">
    Review and approve the permissions Gridix requests:
    - Read pipelines and statuses
    - Create leads and contacts
    - Update lead information
  </Step>
  
  <Step title="Confirm Access">
    Click "Allow" or "Authorize" to grant access.
  </Step>
</Steps>

### Step 3: Configure Integration

After authorization, you'll return to Gridix to configure:

<Steps>
  <Step title="Select Pipeline">
    Choose which AmoCRM pipeline to use for leads:
    - Select from dropdown
    - Pipeline is required
  </Step>
  
  <Step title="Set Status (Optional)">
    Choose initial status for new leads:
    - Select from available statuses
    - Leave blank to use pipeline default
  </Step>
  
  <Step title="Assign Responsible (Optional)">
    Set responsible user for leads:
    - Select from your AmoCRM users
    - Leave blank for automatic assignment
  </Step>
  
  <Step title="Save Configuration">
    Click "Save Settings" to complete setup.
  </Step>
</Steps>

<Check>
Integration is now active! New leads will automatically be sent to AmoCRM.
</Check>

## Integration Configuration

### Pipeline Settings

Configure which pipeline receives leads:

- **Primary Pipeline**: Main pipeline for leads
- **Pipeline Selection**: Choose from available pipelines
- **Pipeline Information**: View pipeline details

<Tip>
Choose a pipeline that matches your sales process. Leads will be created in the selected pipeline.
</Tip>

### Status Configuration

Set the initial status for new leads:

- **Default Status**: Use pipeline default
- **Custom Status**: Select specific status
- **Status Mapping**: Map Gridix statuses to AmoCRM statuses

### Responsible User

Assign leads to specific users:

- **Auto Assignment**: AmoCRM's automatic assignment
- **Specific User**: Assign to a particular user
- **Round Robin**: Distribute leads evenly

<Info>
Leads are assigned to the responsible user when created in AmoCRM.
</Info>

## How It Works

### Automatic Lead Submission

<Steps>
  <Step title="Lead Created in Gridix">
    Customer submits inquiry through your floor plan widget.
  </Step>
  
  <Step title="Integration Triggered">
    Gridix detects new lead and prepares for CRM submission.
  </Step>
  
  <Step title="Data Preparation">
    Lead data is formatted for AmoCRM:
    - Customer information
    - Apartment details
    - Project information
  </Step>
  
  <Step title="CRM Submission">
    Lead is sent to AmoCRM via API.
  </Step>
  
  <Step title="Status Update">
    Gridix updates lead status to "Sent to CRM" upon success.
  </Step>
</Steps>

### Lead Data Mapping

Gridix sends the following data to AmoCRM:

  <Card title="Lead Details">

    - Apartment number
    - Apartment details
    - Price information
  </Card>
</CardGroup>

## Managing Integration

### Updating Settings

<Steps>
  <Step title="Open Integration Settings">
    Navigate to CRM Integration section.
  </Step>
  
  <Step title="Edit Configuration">
    Click "Edit Settings" or "Change Settings".
  </Step>
  
  <Step title="Update Options">
    Modify:
    - Pipeline selection
    - Status assignment
    - Responsible user
  </Step>
  
  <Step title="Save Changes">
    New settings apply to future leads.
  </Step>
</Steps>

### Reconnecting

If connection is lost:

<Steps>
  <Step title="Check Connection Status">
    View integration status in settings.
  </Step>
  
  <Step title="Reconnect">
    Click "Reconnect" or "Update Connection".
  </Step>
  
  <Step title="Re-authorize">
    Complete AmoCRM authorization again.
  </Step>
</Steps>

### Disconnecting

To disconnect AmoCRM:

<Warning>
Disconnecting will stop automatic lead submission. Existing leads remain in Gridix.
</Warning>

<Steps>
  <Step title="Open Integration Settings">
    Navigate to AmoCRM integration.
  </Step>
  
  <Step title="Disconnect">
    Click "Disconnect" button.
  </Step>
  
  <Step title="Confirm Disconnection">
    Confirm the action. Integration is disabled.
  </Step>
</Steps>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Leads not sending to AmoCRM">
    - Verify integration is connected
    - Check pipeline selection is valid
    - Ensure API access is active
    - Review error logs in integration settings
  </Accordion>
  
  <Accordion title="Integration disconnected">
    - Reconnect AmoCRM
    - Verify AmoCRM account is active
    - Check API permissions
    - Contact support if issue persists
  </Accordion>
  
  <Accordion title="Wrong pipeline or status">
    - Update integration settings
    - Select correct pipeline
    - Choose appropriate status
    - Save changes
  </Accordion>
  
  <Accordion title="Leads appear in wrong status">
    - Review status mapping
    - Update status configuration
    - Check AmoCRM pipeline settings
    - Verify responsible user assignment
  </Accordion>
</AccordionGroup>

### Error Handling

Gridix handles errors gracefully:

- **Automatic Retry**: Failed submissions are retried automatically
- **Error Logging**: Errors are logged for review
- **Status Tracking**: Failed leads show "Failed" status
- **Manual Retry**: Manually retry failed submissions
